## Part A - Execution Instructions and Key Results (AA subset only)

### Objective

Repeat Problem 1 in Apache Spark based on the provided `SparkWordCount.scala` style:

- (1) Find all individual words (not numbers) with total count exactly `1000`
- (2) Find all word pairs (not numbers) with total count exactly `1000`, using ordered forward neighbors with distance `m <= 2`
- (3) Find the top-100 most frequent individual words (not numbers)
- (4) Find the top-100 most frequent `number:word` pairs, again with `m <= 2`

### Dataset scope

- Input path: `./Wikipedia-En-41784-Articles/AA`
- Number of input files: `100`

### Solution strategy

- Implemented in one Spark application/class: `SparkProblem2`
- Based on `SparkConf`, `SparkContext`, recursive input scan, and `sc.textFile(inputPath)`
- No processing outside Spark is used for the actual task logic
- All counting is done with `reduceByKey`
- Exact top-100 results are computed directly in Spark
- Pair generation is done inside each line after token filtering

### Token and pair definition clarification

- Each line is first converted to lower case
- The tokenizer splits on non-word separators
- Valid word tokens:
  - lower-case `a-z` and `-`
  - length `6..24`
- Valid number tokens:
  - digits `0-9` and `.`
  - optional leading `-`
  - length `4..16`
- Pair generation is ordered and forward-only inside each line after token filtering
- For token index `i`, neighbors are `i+1` and `i+2` if present, so `m in {1,2}`
- No backward pairs are emitted
- No pairs are emitted across different lines
- For Task (4), only pairs matching `number:word` in that exact order are counted

### Build commands (from repository root)

Compile Scala source:

```bash
scalac -classpath "$(echo "$SPARK_HOME"/jars/*.jar | tr ' ' ':')" -d problem_2/out problem_2/src/SparkProblem2.scala
```

Create/recreate jar:

```bash
jar -cvf problem_2/SparkProblem2.jar -C problem_2/out .
```

### Run commands (AA only)

```bash
rm -rf problem_2/test_output/problem2_full_AA
/usr/bin/time -p spark-submit --master local[*] --class SparkProblem2 problem_2/SparkProblem2.jar ./Wikipedia-En-41784-Articles/AA problem_2/test_output/problem2_full_AA
```

### Output format (`part-00000`)

- `TASK3\t<word>\t<count>` for (3) top-100 words
- `TASK4\t<number:word>\t<count>` for (4) top-100 number-word pairs

### Run commands

Run the full cumulative benchmark with the provided script:

```bash
cd problem_2
./run_problem2_scalability.sh
```

### What the script does

The script automates the full scalability experiment. It:

- iterates cumulatively through `AA`, `AB`, ..., `AK`
- builds the corresponding Spark input path list for each step
- counts how many input files are included at that step
- removes any previous output directory for that step
- runs one timed `spark-submit` execution
- stores the raw timing output in `benchmark_logs/<LABEL>.time`
- counts how many lines belong to `TASK1`, `TASK2`, `TASK3`, and `TASK4`
- writes the final summary table to `benchmark_logs/problem2_scalability.tsv`

This avoids running each cumulative case manually.
